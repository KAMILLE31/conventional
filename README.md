# 📖 JW LIBRARY (Light Web App Edition)

오리지널 JW Library 앱의 핵심 UX/UI와 레이아웃을 순수 웹 표준 기술(HTML5, CSS3, Vanilla JavaScript)로 정밀하게 재현한 초경량 모바일 최적화 웹 애플리케이션입니다. 

네트워크 지연이나 리소스 낭비 없이 빠르고 쾌적하게 성경 목록 탐색, 장(Chapter) 선택, 그리고 세련된 명조체 기반의 성경 리더 뷰를 이용하실 수 있습니다.

---

## ✨ 핵심 기능 (Features)

1. **순정 UI 완벽 오마주**
   - 상단 액션바 및 하단 네비게이션 탭의 고유 비주얼 구현.
   - 성경 약어 그리드(He/Gr) 분할 구성 및 장 선택 패널 제공.
2. **지능형 듀얼 성경 데이터 로더 (중요)**
   - **GitHub CDN 자동 연동:** 원격 서버에 동기화된 성경 데이터를 터치 한 번으로 즉시 불러옵니다.
   - **샌드박스 로컬 수동 업로드:** 오프라인 상태이거나 서버에 없는 개인 자료(TXT)를 앱 내부 메모리에 안전하게 구조화하여 즉석에서 읽을 수 있습니다.
3. **PWA 최적화 및 네이티브 하드웨어 뒤로가기 제어**
   - 모바일 환경 및 홈 화면 추가 상태에서 하단 하드웨어/제스처 '뒤로가기'를 클릭하더라도 앱이 종료되지 않고 직전 메뉴(독서 뷰 ➡️ 장 선택 ➡️ 메인 목록)로 유기적으로 상위 라우팅됩니다.
4. **고급 인터랙션 및 개인화**
   - **풀다운 제스처 내비게이션:** 리더 스크린 상단/하단을 스와이프 업/다운하여 당기면 부드러운 트랜지션 애니메이션과 함께 즉시 이전 장 / 다음 장으로 넘어갑니다.
   - **롱프레스 북마크 기능:** 원하는 성경 구절을 길게 누르면 북마크 상태가 하이라이트 처리되며, 로컬스토리지에 저장되어 재접속 시에도 유지됩니다.

## 🗂 성경 데이터(.txt) 제공 및 업로드 방식 안내

본 애플리케이션은 사용자의 편의를 위해 **2가지 데이터 로드 방식**을 지원합니다. 성경 데이터를 준비할 때는 파일명이 반드시 **성경 영문 약어(예: `Ge.txt`, `Mt.txt`)**와 일치해야 합니다.

### 1️⃣ 방식 A: GitHub 원격 자동 업로드 (권장)
개발자 또는 관리자가 GitHub 저장소에 성경 파일을 미리 올려두어, 사용자가 앱에서 책 버튼을 누르는 순간 자동으로 데이터를 다운로드하는 방식입니다.

* **연동 원리:** JavaScript 내부의 `CLOUD_BASE_URL` 경로를 기반으로 파일(예: `Ge.txt`)을 비동기(`fetch`) 호출합니다.
* **적용 방법:**
  1. 본인의 GitHub 레포지토리(예: `main` 브랜치)에 성경 약어로 된 TXT 파일들을 업로드합니다.
  2. 소스코드 상단의 `CLOUD_BASE_URL` 변수 주소를 본인의 깃허브 raw 주소(`https://raw.githubusercontent.com/유저명/레포지토리명/refs/heads/main/`)로 수정합니다.
  3. 이제 사용자는 별도의 파일 선택 없이 버튼 클릭만으로 성경을 즉시 읽을 수 있습니다.
  4. >파일 제목은 버튼에 있는 성경책 약어로 해야함. 예를들어 Ge(Genesis)버튼에 연동하려면 'Ge.txt'로 업로드

### 2️⃣ 방식 B: 로컬 파일 수동 업로드
원격 서버(GitHub)에 데이터가 없거나 인터넷 연결이 불가능한 환경에서 사용자가 가지고 있는 로컬 단말기의 TXT 파일을 직접 앱에 주입하는 방식입니다.

* **동작 시점:** GitHub 서버에서 데이터를 가져오는 데 실패(`404 에러` 또는 네트워크 단절)하면, 시스템이 자동으로 단말기의 **파일 선택 창**을 활성화합니다. 또는 상단 우측 더보기(`...`) 메뉴의 **'로컬 파일 수동 업로드'**를 눌러 원할 때 언제든 파일을 갱신할 수 있습니다.
* **데이터 보안:** HTML5 `FileReader API`를 사용하여 사용자의 기기 내부에서만 텍스트를 파싱하므로, 외부 서버로 데이터가 절대 전송되지 않고 안전합니다.
* **텍스트 파일 규격 주의사항:**
  - 성경 데이터는 행바꿈(`\n`)을 기준으로 하나의 절(Verse)을 파싱합니다. 각 행이 한 절이 되도록 정렬된 텍스트 파일이 가장 정갈하게 렌더링됩니다.
  - 내장 텍스트 파서 엔진이 작동하여 본문 내 한글 형태소가 검출되면, 영문 고딕체에서 선명하고 부드러운 **KoPubWorld 바탕체(Medium 500)** 스타일로 즉시 자동 스위칭됩니다.

## 🛠 확장 개발을 위한 인터페이스(더미 버튼) 안내
현재 상/하단바 및 서브 탭바에 인터페이스 구색을 위해 배치된 기능 버튼들은 사용자 안내를 위한 `alert()` 처리 상태이며, 향후 다음과 같이 기능을 확장할 수 있습니다.

* **돋보기(검색) 버튼:** 현재 메모리에 로드된 성경 데이터를 정규식(`RegExp`)으로 스캔하여 단어 통합 검색창(Modal)으로 확장.
* **북마크 버튼:** `localStorage` 내부의 하이라이트 데이터를 모아 보여주는 '나의 구절 저장소' 기능 연결.
* **INTRODUCTION, INDEX, APPENDICES 탭:** 성경 서론 정보 기술 및 도량형 환산표, 성경 지도 팝업 뷰어 배치 공간으로 활용.
* **하단 내비게이션 바 (홈, 라이브러리, 집회, 개인 연구):** 오늘의 성구 대시보드, 추가 서적용 파일 매니저, 주간 집회 스케줄 및 연구 노트 기록실로 확장 가능.
